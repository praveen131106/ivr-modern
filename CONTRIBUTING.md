# Contributing Guidelines

Thank you for your interest in contributing to the Train IVR System project!

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable names
- Add comments for complex logic
- Maintain consistent formatting

## Adding New Flows

1. Create a new JSON file in `backend/flows/`
2. Define states, transitions, and actions
3. Add flow to `flow_manager.py` flow list
4. Update documentation

## Testing

- Test all keypad inputs
- Test voice commands
- Verify error handling
- Check session management

## Pull Request Process

1. Fork the repository
2. Create a feature branch
3. Make your changes
4. Test thoroughly
5. Submit pull request with description
